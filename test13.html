    5. 対応デバイス（PC/スマホ/タブレット）：
    6. 言語対応：

  2. フォーム構成
    1. 入力項目一覧
      1. 個人情報（氏名、住所、電話番号、メールアドレス、生年月日 など）
      2. 保険情報（保険種別、補償範囲、希望プラン など）
      3. 決済情報（クレジットカード、銀行振込 など）
      4. 同意事項（利用規約、プライバシーポリシー など）

    2. 入力項目のバリデーション（必須入力、文字種、桁数制限 など）
    3. フォームステップ（例：基本情報 → 保険選択 → 決済 → 確認・送信）
    4. 入力補助（オートコンプリート、選択式リスト、エラーメッセージ など）
    5. 送信後の処理（完了画面、確認メール送信 など）

  3. データ連携・管理
    1. 申込データの保存方法（DB設計、ログ管理）
    2. 外部システム連携（決済ゲートウェイ、顧客管理システム など）
    3. API利用の有無・仕様
    4. セキュリティ対策（個人情報の暗号化、アクセス制限 など）

  4. UI/UX設計
    1. フォームのデザイン方針（シンプル、視認性向上、ブランドカラー など）
    2. ユーザビリティの考慮（入力負担軽減、誤入力防止 など）
    3. モバイル対応（レスポンシブデザイン、タップしやすいUI など）

  5. テスト・運用
    1. テスト項目（動作確認、入力チェック、デバイス対応確認 など）
    2. エラー発生時の対応（エラーメッセージ、リカバリフロー）
    3. 継続的な改善・メンテナンス方針
